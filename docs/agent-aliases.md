# Agent Alias Policy

## Purpose

This document records natural-language alias expectations for agent routing.

## Current Aliases

### `angel`

Route these forms to `angel`:

- `angel`
- `엔젤`
- `천사`

## Planned Aliases

### `devil`

## Notes

- Alias handling should remain lightweight and practical.
- Descriptions should include the most common aliases when it helps routing.
- Natural-language routing rules should live in the installed global `AGENTS.md`.
