# Session Notes - 2026-03-19

## Summary

This session focused on rebuilding the first usable Codex harness layer from the old `claude-automate` ideas.

The current direction is:

- global setup first
- project-local state later
- `AGENTS.md` as a routing/indexing document
- custom Codex agents for exploration, implementation, and thought expansion

## Decisions Made

### Setup Model

- `codex-automate` installs into `~/.codex/`
- explicit project `init` is not part of the current V0 direction
- `setup` backs up existing files before replacing managed ones

### AGENTS.md Policy

- `AGENTS.md` should be treated as an indexing and routing document whenever possible
- detailed rules should live in separate focused documents
- this policy applies both to the repository itself and to the installed global `~/.codex/AGENTS.md`

### Delegation Rules

- code exploration should route to the `explorer` family
- code writing and any file-writing task should route to the `worker` family
- non-implementation agents should default to `read-only`
- implementation agents should use `workspace-write`

### Custom Agent Direction

The built-in Codex roles were not considered enough for the preferred workflow style, so custom roles were chosen.

Current agent plan:

- `explorer`
- `explorer_deep`
- `worker`
- `worker_high`
- `angel`
- `devil`
